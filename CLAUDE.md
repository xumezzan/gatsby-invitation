# CLAUDE.md — память проекта

## Что это
Одностраничный веб-пригласительный «Latifa's Great Birthday Celebration» в стиле Great Gatsby / арт-деко.
Событие: 12.07.2026, зал Miras, Ташкент. Рассылается гостям ссылкой через Telegram → основной сценарий просмотра — мобильный.

## Главный принцип: один файл
Весь сайт живёт в `index.html`: разметка, стили, JS; все текстуры — inline SVG data-URI (feTurbulence) либо процедурные (WebGL-шейдер).
Деплой — Netlify Drop. **Не выносить CSS/JS/картинки в отдельные файлы без прямой просьбы владельца** — это сломает его workflow «перетащил — сайт работает».
Исключения рядом с index.html (деплоятся ТОЙ ЖЕ папкой, тащить в Netlify вместе): `og-image.png` (превью Telegram не умеет data-URI), `atmos.mp4` (видео нельзя в base64 — раздует HTML), опциональный `music.mp3`.
CDN-зависимости (решение владельца, 07.2026, версии пиновые): Three.js r128 UMD + GSAP 3.12.5 с cdnjs — только для интро. **Graceful degradation обязателен**: если CDN/WebGL недоступны, интро обязано работать в режиме `no-fx`.

## Команды
- Локально: открыть `index.html` или `npx serve .` / `python3 -m http.server`
- Сборки нет. Зависимостей нет. Линтеров нет. Не добавлять package.json без запроса.

2. `#intro` — кинематографичное интро «Black Velvet × Champagne Gold» (WebGL-канвас `#veil`, тиснение `#introName`, текст-кнопка `#enterBtn`) + оверлей `#glass` (дымчатое стекло перехода)
3. `<defs>` с градиентами `#gild`/`#gildH` и уголком `#cornerDeco` — сразу после `<body>` (используются по всему документу)
4. `.hero` — экран-афиша «12 JULY 2026 · MIRAS» (канвас `#sparkles` с боке, `.screen2`: очень крупная editorial-дата + countdown одной строкой без коробок; секций Countdown/Details больше НЕТ — их содержимое здесь)
5. `.atmos` — экран 3 «Атмосфера вечера»: полноэкранное фоновое ВИДЕО `<video class="atmos-bg">` (налив шампанского, `atmos.mp4` рядом) с параллаксом и veil-градиентом
6. Dress code — фешн-галерея `.looks` (2 крупных фото образов в дуотоне, вертикальные подписи For Her/For Him, параллакс) + палитра Black/Gold/Silver

- Палитра в `:root`: `--gold #d4af37`, `--gold-soft #f4e3ae`, `--gold-deep #8f6f22`, `--silver #cdd2da`, фон `#0a0a0c`

## Критические паттерны — НЕ ЛОМАТЬ

### 1. Script-шрифт + background-clip:text = обрезка глифов
Градиент красится только внутри бокса элемента; выносные элементы Great Vibes (росчерк L, апостроф, хвост f) выходят за бокс и «стираются».
**Правило: у любого элемента с Great Vibes + `.gold-text` обязательны `line-height ≥ 1.3` и `padding` по горизонтали.**
Уже применено к `.hero-name`, `.finale-see`, `.intro-name`. `.hero-great` (Italiana) держать на `line-height ≥ 1.04`.

### 2. Размеры текста в интро — только `min(vw, rem)`
На Android с увеличенным системным шрифтом браузер раздувает `rem`; `clamp()` с rem-минимумом делал текст шире экрана (баг воспроизводился, чинили).
**Правило: в интро никаких rem-минимумов в font-size. Только `min(Xvw, Yrem)`.** Плюс `.intro-content > * { max-width:100% }`.

### 3. Интро «Black Velvet × Champagne Gold» (двери удалены — редизайн по спеке владельца 07.2026)
- WebGL-бархат: Three.js ShaderMaterial на `#veil` — анизотропный fbm (вертикальная драпировка), световой проход (`uSweep`), «дышащий» прожектор, дым, живое зерно. DPR cap 1.5.
- Сцены (GSAP-таймлайн `introFX.seq`, темп сжат 07.2026 — владелец: вступление было слишком долгим): 0.1s проявление (`uGlow`, 1.0s) → 0.2s свет по бархату (`uSweep`, 1.6s) → 0.8s тиснение LATIFA (opacity + сужение letter-spacing, класс `.foil` в 1.5s даёт разовый металлический блик) → 1.4s You're invited → 1.8s кнопка. Полная длина seq ~2.5s (было ~4.4s). Дальше сжимать нежелательно — свет по бархату не успеет прочитаться.
- Выход по клику (`introFX.out`) — **занавес раздвигается** (редизайн 07.2026, владелец: золотая пыль/THREE.Points выглядела дёшево, удалена НАВСЕГДА — не возвращать): uniform `uSplit` 0→1 (`power3.inOut`, 1.5s) двигает семплирование ткани наружу (полы бархата уезжают к краям), в центре ширится золотой световой проём (тёплый градиент снизу вверх + подсвеченный кант `rim` по кромке полы). Имя тает в свет проёма (opacity+blur+чуть вверх, 0.7s). `uZoom` наезд 1→1.06, `#glass` (backdrop-filter blur — «дымчатое стекло», z-index 950) вспыхивает на пике раскрытия и гаснет, интро фейдится и удаляется по `onComplete`. Ключевые части шейдера: `open`/`onCloth` (граница ткань/проём), `ps` (сдвинутые коорд. семплирования ткани — складки/ворс/отблеск идут по `ps`, прожектор/дым/виньетка по экранному `p`). ВНИМАНИЕ: `half` — зарезервированное слово GLSL, переменную звать `halfW`.
- **Fallback `no-fx` обязателен**: нет THREE/gsap/WebGL или reduced-motion → класс `no-fx` (CSS fadeUp-появление), клик → `.open` (CSS-фейд .8s) + `setTimeout(remove, 900)`. Проверять оба пути.
- Кнопка `.open-inv` — ЯВНАЯ обрамлённая кнопка: золотая рамка + внутренняя деко-линия (`::before`) + заливка + пульсирующее свечение (`invitePulse`, box-shadow, стартует ~4с после появления) + hover/active. (Была тонкая текст-строка с бликом — владелец попросил сделать очевидно кликабельной, 07.2026. Прежний вариант не возвращать.) Пульс дублируется в `#intro.no-fx .open-inv` через запятую вместе с fadeUp; в WebGL-пути opacity ведёт GSAP, а box-shadow-пульс из CSS — не конфликтуют.
- Headless-тесты: часы GSAP не виртуализируются (`--virtual-time-budget` двигает rAF, но не Date.now) — таймлайн в скриншот-тестах перематывать вручную: `introFX.seq.progress(x, false); gsap.ticker.tick();`. WebGL в headless shell требует флагов `--enable-unsafe-swiftshader --use-angle=swiftshader`.

### 4. Дресс-код — фешн-галерея (этап 4, фото владельца)
`.looks`: две `<figure class="look">` → `.look-photo` (overflow:hidden, aspect 3/4) → `.look-shift` (слой параллакса) → `<img>`. Плюс `.look-vert` (вертикальная подпись writing-mode) и `.look-cap` (Italiana-заголовок + Cinzel-описание). Фото — СОБСТВЕННЫЕ фото владельца (флэппер-платье / смокинг, уже в тёплом золоте на чёрном — дуотон НЕ накладывать, только ресайз 860px + оптимизация). Замена: поменять `src` у `<img>`. object-fit:cover, object-position:center 30% (полный рост, ноги слегка кроп). Копии в `assets/dresscode-*.jpg`.
Hover-зум: `.look:hover img{ transform:scale(1.07) }` под `@media (hover:hover)` (touch не залипает). transform на `<img>` свободен — параллакс двигает `.look-shift`, не img, иначе inline-transform от JS перебил бы CSS-hover.

### 5. Reduced motion
В конце CSS глобальный киллер `*{animation:none; transition:none}` под `prefers-reduced-motion` + отдельный медиа-блок, который выставляет интро финальные состояния (`opacity:1`, `stroke-dashoffset:0`, `#intro.open{display:none}`). Любая новая анимация с `opacity:0` в стартовом состоянии обязана получить переопределение в этом блоке, иначе у части гостей элемент останется невидимым.

### 6. Музыка
`bgm.play()` вызывается только в обработчике клика `#enterBtn` (autoplay-политика браузеров). Файл `music.mp3` кладётся рядом с index.html; если его нет — `catch` глотает ошибку, кнопка звука не показывается, сайт работает. Громкость 0.35. Лицензионные треки не вшивать — только royalty-free.

### 7. Канвасы
Три независимых канваса: `#sparkles` (hero, 2D-боке: три плана глубины, пре-рендеренные спрайты `drawImage`, DPR cap 2), `#veil` (интро, WebGL — см. паттерн №3, DPR cap 1.5), `#finaleSmoke` (финал, 2D дым+блики, DPR cap 1.5, троттлинг 30 к/с). Все рисуют с учётом `devicePixelRatio` (`ctx.setTransform(DPR,...)`) — без этого на телефонах мыло. **Оба 2D-канваса (`#sparkles` И `#finaleSmoke`) gated через IntersectionObserver** (running=true/false, cancelAnimationFrame) — рисуют ТОЛЬКО пока своя секция в зоне видимости, освобождают главный поток и берегут батарею (правка 07.2026: в Telegram-WebView рвался финал — вечно крутившийся hero-боке конкурировал за главный поток; плюс дым additive-`lighter` с 10 большими плюмами при DPR 2/60fps перегружал fillrate → снижены до DPR 1.5 + троттлинг 30 к/с, вид не изменился — дым медленный и размытый). `#finaleSmoke`: rAF планируется в НАЧАЛЕ кадра, троттлинг только пропускает отрисовку (`if(ts-lastFrame<FRAME_MS) return`), `lastFrame=-1000` чтобы первый (в т.ч. reduced-motion) кадр гарантированно рисовался.
Дым: additive (`globalCompositeOperation='lighter'`) крупные размытые плюмы, радиус адаптивен `p.rf * Math.max(W,640)` — на узком экране не сливаются в сплошную дымку (баг ловили: фикс-радиус 300px на 380px-ширине давал коричневую заливку вместо чёрного).
ВНИМАНИЕ при headless-тестах: весь клиентский JS — в ОДНОМ `<script>`; если изолировать одну секцию, отсутствие DOM-элементов (`#sparkles`,`.hero`,`#enterBtn`,`cd-*`) роняет скрипт до нужного места (`null.getContext`). Для теста секции добавлять скрытые заглушки всех этих id, иначе «канвас не рисует» — ложный вывод.

### 8. SVG-градиенты на линиях: только #gildH
`#gild` (objectBoundingBox) на строго горизонтальных/вертикальных `<path>` НЕ РИСУЕТСЯ вовсе — у линии нулевой bounding box (баг уже ловили: разделитель исчезал). Для линий использовать `#gildH` (`gradientUnits="userSpaceOnUse"`), для заливок фигур — `#gild`. Оба определены в `<defs>` внутри hero-карточки и работают по всему документу.

### 9. Золото = материал, а не заливка
`.gold-text` — два слоя: фольга-текстура (SVG feTurbulence data-URI, blend overlay) + градиент. Блик двигается через `--sheen`; «пробег» блика — одноразовые анимации `sheenSweep` (по `.reveal.in` и `body.lit`, который вешается кликом Enter). **Не возвращать бесконечный shimmer** — вечное мигание выглядит дёшево, убрали осознанно.

### 10. Кнопка музыки и [hidden]
У `.music-toggle` авторский `display:grid` перебивает браузерный `[hidden]{display:none}` — правило `.music-toggle[hidden]{display:none}` обязательно, иначе кнопка видна и без music.mp3 (баг уже чинили).

### 11. RSVP — встроенная форма → Google Таблица через Apps Script (без бэкенда)
`#rsvpForm` в `.finale`: поле имени + radiogroup «буду/не смогу» → отправка на веб-приложение Apps Script `fetch(RSVP_FORM, {method:'POST', mode:'no-cors', body:URLSearchParams})`, которое `doPost` дописывает строку **Дата · Имя · Присутствие** в связанную Google Таблицу. Бэкенда НЕТ, принцип «один файл» сохранён (вся форма и JS inline). Google Форма-посредник УБРАНА (владелец 07.2026: хотел «просто таблицу»); полный код скрипта (`doPost` + `ensureHeader` + `setupSheet`) — в README раздел «RSVP». Скрипт при первом ответе на пустом листе сам ставит шапку `Дата · Имя · Придёт` и закрепляет строку 1 (`ensureHeader`); `setupSheet` — ручной запуск для оформления сразу.
- **3 константы** вверху блока `RSVP → GOOGLE ТАБЛИЦА (Apps Script)` в `<script>`: `RSVP_FORM` (URL веб-приложения, оканчивается на `/exec`, НЕ `/macros/library/…`), `RSVP_YES`/`RSVP_NO` (тексты, попадающие в колонку «Присутствие» — `Да`/`Нет`). Больше НЕ `entry.XXXX` (это было для Google Формы). Владелец публикует скрипт и вписывает URL сам (нет доступа к его Google-аккаунту).
- **Тело = `URLSearchParams`, НЕ `FormData`**: URLSearchParams даёт `application/x-www-form-urlencoded` — Apps Script надёжно кладёт его в `e.parameter.name`/`e.parameter.attend`; content-type CORS-safe для no-cors. FormData шлёт multipart, который Apps Script парсит иначе (проверено: urlencoded точно работает).
- **`no-cors` = opaque-ответ**: fetch не видит статус, поэтому успех = факт отправки (`.then(finish)`); в `.catch` попадает только реальный обрыв сети (тогда разблокируем кнопку, `sent=false`). Не пытаться читать `response.ok` — он недоступен. При успешном POST Apps Script отвечает 302-редиректом на `/macros/echo?...&lib=...` — это НОРМА (строка уже записана до редиректа); `lib=` в адресе и curl-405 на финальном хопе НЕ означают ошибку (ловили это заблуждение при отладке).
- **Guard плейсхолдера**: если `RSVP_FORM` не начинается с `http` (константа не заполнена) — НЕ шлём запрос, показываем `rsvpErrNet` + `console.warn`. Иначе задеплоенная с плейсхолдером форма молча слала бы мусор.
- **Частая ошибка деплоя**: если при публикации выбран тип «Библиотека» вместо «Веб-приложение» — URL получается `/macros/library/d/…`, POST не исполняет `doPost`. Нужен именно тип «Веб-приложение», доступ «Все», при правках кода — «Новая версия» деплоя.
- **i18n placeholder**: поле имени использует `data-i18n-ph` (не `data-i18n`) → `applyLang` ставит `placeholder`. Тексты/ошибки/благодарности — ключи `rsvp*` в обоих словарях; сообщения берутся по текущему `LANG` в момент отправки.
- CSS: выбранный вариант подсвечивается через `.rsvp-opt:has(input:checked)` (нативный radio скрыт `opacity:0`). Успех → класс `.done` на форме прячет поля/кнопку, остаётся благодарность. Reduced-motion безопасно: форма стартует с `opacity:1` (не появляется из 0), только `.rsvp-status` скрыт до отправки — это состояние, не анимация.
